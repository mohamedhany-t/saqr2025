# Prompt تفصيلي لنظام الصقر للخدمات اللوجستية (AlSaqr Logistics System)

## 1. نظرة عامة على المشروع

**اسم المشروع:** نظام الصقر للخدمات اللوجستية | AlSaqr Logistics System

**الغرض الأساسي:** هو نظام متكامل لإدارة عمليات الشحن والتوصيل في مصر، مصمم ليكون سريعًا، فعالًا، وقابلًا للتطوير. يدعم النظام تعدد المستخدمين بأدوار وصلاحيات مختلفة، ويوفر مجموعة قوية من الأدوات التي تجعل إدارة الشحنات، الحسابات المالية، والفرق الميدانية عملية سهلة ومنظمة.

## 2. الهيكل التقني (Tech Stack)

- **إطار العمل (Framework):** Next.js (مع استخدام App Router).
- **لغة البرمجة:** TypeScript.
- **واجهة المستخدم (UI):** React، مكونات ShadCN UI، وتصميم Tailwind CSS.
- **الخلفية وقاعدة البيانات (Backend & DB):** Google Firebase.
  - **المصادقة (Authentication):** Firebase Authentication (بريد إلكتروني وكلمة مرور).
  - **قاعدة البيانات:** Firestore NoSQL Database.
- **قدرات الذكاء الاصطناعي (AI):** Genkit (خاصة لتدفقات العمل الذكية مثل تعيين المناديب).
- **التصميم (Styling):** يتم التحكم في المظهر الرئيسي من خلال ملف `src/app/globals.css` الذي يستخدم متغيرات HSL CSS.

## 3. نماذج البيانات الأساسية (Core Data Models)

هذه هي المجموعات (Collections) الرئيسية في قاعدة بيانات Firestore:

### أ. `users`
- **الوصف:** يخزن معلومات جميع المستخدمين في النظام.
- **الحقول:**
  - `id` (string): معرف المستخدم من Firebase Auth (UID).
  - `name` (string): اسم المستخدم (أو اسم الشركة).
  - `email` (string): البريد الإلكتروني.
  - `role` (string): دور المستخدم (`admin`, `company`, `courier`).
  - `companyId` (string): (اختياري) معرّف الشركة.
  - `createdAt` (Timestamp): تاريخ إنشاء الحساب.
  - `commissionRate` (number): (خاص بالمناديب) عمولة ثابتة لكل شحنة.

### ب. `shipments`
- **الوصف:** يمثل سجلات الشحنات الفردية.
- **الحقول:**
  - `id` (string): معرف فريد للشحنة.
  - `shipmentCode` (string): كود فريد للشحنة (مثال: `SH-20240728-0001`).
  - `orderNumber` (string): رقم الطلب الخاص بالعميل.
  - `trackingNumber` (string): رقم التتبع.
  - `recipientName` (string): اسم المستلم.
  - `recipientPhone` (string): هاتف المستلم.
  - `governorateId` (string): معرّف المحافظة.
  - `address` (string): عنوان المستلم التفصيلي.
  - `status` (string): حالة الشحنة (Enum: `Pending`, `In-Transit`, `Delivered`, `Partially Delivered`, `Returned`, `Cancelled`, `Postponed`, `Returned to Sender`, `Refused (Paid)`, `Refused (Unpaid)`, `Evasion (Phone)`, `Evasion (Delivery Attempt)`).
  - `totalAmount` (number): المبلغ الإجمالي المطلوب تحصيله.
  - `paidAmount` (number): المبلغ الذي تم تحصيله بالفعل.
  - `courierCommission` (number): عمولة المندوب المحسوبة لهذه الشحنة.
  - `companyCommission` (number): عمولة النظام المحسوبة على الشركة لهذه الشحنة.
  - `companyId` (string): معرّف الشركة (العميل الرئيسي) المالكة للشحنة.
  - `assignedCourierId` (string): معرّف المندوب المكلف بالشحنة.
  - `isArchivedForCourier` (boolean): علامة لأرشفة الشحنة من حساب المندوب بعد التسوية.
  - `isArchivedForCompany` (boolean): علامة لأرشفة الشحنة من حساب الشركة بعد التسوية.

### ج. `companies`
- **الوصف:** يخزن معلومات الشركات (العملاء الرئيسيين).
- **الحقول:**
  - `id` (string): المعرف الفريد (نفس UID المستخدم).
  - `name` (string): اسم الشركة.
  - `governorateCommissions` (map): خريطة تحتوي على عمولات النظام لكل محافظة (مثال: `{ "gov_cairo": 5, "gov_giza": 7 }`).

### د. `couriers`
- **الوصف:** يخزن معلومات المناديب.
- **الحقول:**
  - `id` (string): المعرف الفريد (نفس UID المستخدم).
  - `name` (string): اسم المندوب.
  - `commissionRate` (number): العمولة الثابتة لكل شحنة ناجحة أو مرتجعة.

### هـ. `governorates`
- **الوصف:** قائمة المحافظات المتاحة في النظام.
- **الحقول:**
  - `id` (string): معرف فريد.
  - `name` (string): اسم المحافظة (باللغة العربية).

### و. المدفوعات (`courier_payments`, `company_payments`)
- **الوصف:** سجلات لجميع الدفعات المالية التي تتم لتسوية الحسابات.
- **الحقول:**
  - `id` (string): معرف فريد لعملية الدفع.
  - `courierId` / `companyId` (string): معرّف المندوب أو الشركة.
  - `amount` (number): المبلغ المدفوع.
  - `paymentDate` (Timestamp): تاريخ تسجيل الدفعة.
  - `recordedById` (string): معرّف المسؤول الذي سجل الدفعة.
  - `isArchived` (boolean): علامة لأرشفة الدفعات بعد التسوية الكاملة.

## 4. الأدوار والصلاحيات (Roles & Permissions)

### أ. المسؤول (Admin)
- **الوصول:** رؤية كاملة لكل شيء.
- **الصلاحيات:**
  - إنشاء/تعديل/حذف جميع أنواع المستخدمين (شركات، مناديب).
  - إدارة كاملة للشحنات (إضافة، تعديل، حذف، استيراد/تصدير جماعي).
  - تعيين الشحنات للمناديب يدويًا أو باستخدام مساعد الذكاء الاصطناعي.
  - إدارة مالية شاملة: تسجيل دفعات المناديب والشركات، تسوية وأرشفة الحسابات.
  - الوصول إلى جميع التقارير.
  - إدارة المحادثات مع جميع المستخدمين.
  - الوصول إلى "صندوق المشاكل" لمتابعة الشحنات التي تتطلب تدخلاً.

### ب. الشركة (Company)
- **الوصول:** رؤية شحناتها الخاصة فقط.
- **الصلاحيات:**
  - إضافة شحنات جديدة أو استيرادها عبر ملف Excel.
  - عرض إحصائيات وتقارير خاصة بها.
  - تعيين شحناتها إلى المناديب المتاحين في النظام.
  - التواصل المباشر مع المسؤول عبر نظام الدردشة.
  - لا يمكنها حذف الشحنات أو تعديل الحالات المالية بعد تعيينها.

### ج. المندوب (Courier)
- **الوصول:** رؤية الشحنات المكلف بها فقط.
- **الصلاحيات:**
  - تحديث حالة الشحنات التي في عهدته (تم التسليم، مرتجع، مؤجل، إلخ).
  - إدخال المبلغ المحصّل في حالة التسليم الجزئي.
  - عرض ملخص مالي لحسابه وعمولاته.
  - التواصل مع المسؤول عبر الدردشة.
  - مشاركة تقرير سريع للشحنة عبر واتساب.
  - لا يمكنه تعديل تفاصيل الشحنة الأساسية (الاسم، العنوان، المبلغ).

## 5. الميزات والوظائف الرئيسية

### أ. إدارة الشحنات
- **واجهة عرض الشحنات:** جدول قابل للفلترة والبحث والترتيب.
- **نموذج إضافة/تعديل:** نافذة منبثقة (Sheet) لإدارة بيانات الشحنة.
- **التحديثات الجماعية (Bulk Actions):** تحديد مجموعة من الشحنات وتغيير حالتها، تعيين مندوب/شركة، أو أرشفتها بضغطة زر.
- **الاستيراد والتصدير:** استيراد شحنات من ملف Excel وتصدير تقارير مخصصة.
- **طباعة الملصقات (Labels):** طباعة ملصقات تحتوي على QR Code لتسهيل تحديث الحالة عبر المسح.

### ب. الإدارة المالية
- **حسابات المناديب:**
  - حساب آلي للعمولة بناءً على حالة الشحنة (مثلاً، عمولة كاملة للتسليم، نصف عمولة للمرتجع).
  - تسجيل الدفعات المستلمة من المندوب لتحديث رصيده.
  - أرشفة الحسابات لتصفير الرصيد وبدء دورة مالية جديدة.
- **حسابات الشركات:**
  - حساب آلي لعمولة النظام على كل شحنة.
  - تسجيل الدفعات المرسلة للشركة لتسوية حسابها.
- **كشوفات الحسابات:** صفحة مخصصة لعرض كشف حساب تفصيلي لأي مندوب أو شركة.

### ج. أدوات الذكاء الاصطناعي
- **مساعد التعيين الذكي (Genkit Flow):**
  - يحلل الشحنات غير المعينة ويقترح أفضل مندوب لكل منها بناءً على الموقع وعبء العمل الحالي.
  - اسم التدفق: `assignmentFlow`.
- **صندوق المشاكل (Problem Inbox):**
  - واجهة مخصصة للمسؤول لعرض الشحنات التي تحتاج تدخلًا فوريًا (مرتجعات، تأخيرات).

### د. التواصل
- **نظام محادثات مدمج:** تواصل فوري بين المسؤول والمناديب/الشركات.
- **إشعارات (Push Notifications):** إشعارات فورية عند وصول رسائل جديدة أو تعيين شحنات جديدة.

## 6. قواعد العمل الهامة (Business Logic Rules)

- **حساب العمولات:**
  - يتم احتساب عمولة المندوب والشركة فقط عندما تتغير حالة الشحنة إلى حالة نهائية تؤثر على الماليات (مثل `Delivered`, `Partially Delivered`, `Refused (Paid)`, `Evasion (Delivery Attempt)`).
  - الحالات الأخرى (مثل `Pending`, `Cancelled`) تؤدي إلى تصفير الحقول المالية.
- **الأرشفة والتسوية:**
  - عند أرشفة حساب مندوب أو شركة، يتم تسجيل دفعة تسوية تلقائيًا بالمبلغ المتبقي، ثم يتم إخفاء جميع الشحنات والدفعات المنتهية من الحسابات النشطة.
- **الصلاحيات:** المبدأ الأساسي هو "التقييد". لا يُسمح لأي دور بالوصول إلى بيانات أو وظائف لا يحتاجها بشكل مباشر.

هذا الوصف يوفر فهمًا عميقًا للنظام، مما يمكّن أي مطور أو ذكاء اصطناعي من فهم البنية الحالية والبناء عليها بفعالية.
